Grabbable behavior for a widget.

Mouse down events that collide with the widget will “grab” it, calling grab(). While grabbed, each mouse event will call grab_update() until the widget is ungrabbed, i.e., a mouse up event is received (which calls ungrab()).

To customize grabbable behavior, implement any of grab(), grab_update(), or ungrab().

For convenience, the change in mouse position is available through mouse_dyx, mouse_dy, and mouse_dx.

Parameters
is_grabbablebool, default: True

If false, grabbable behavior is disabled.

disable_ptfbool, default: False

If true, widget will not be pulled to front when grabbed.

mouse_buttonMouseButton, default: MouseButton.LEFT

Mouse button used for grabbing.
